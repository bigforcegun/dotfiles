# Agent todo.txt bridge

- Status: research/design context, no implementation yet
- Owner context: chat research on 2026-06-27
- Scope: durable, cross-platform task state for coding agents using todo.txt-compatible plain text plus sidecar context files

## Purpose

Capture the research and design direction for moving agent todo state into a portable plain-text structure. The intended shape is not to replace OpenCode's live `todowrite` scratchpad, but to add a durable persistence layer that agents can import/export at session boundaries.

## Reading order

Read this file first. Read `decisions.md` for the recommended architecture. Read `research.md` for supporting sources and community-practice notes.

## Current recommendation

Use todo.txt as a small task index, not as full agent memory:

1. Canonical task state lives in per-project `todo.txt`/`done.txt` files.
2. Each task line remains human-scannable and todo.txt-compatible.
3. Long context, acceptance criteria, investigation notes, and handoff details live in Markdown sidecars referenced by stable task IDs.
4. Global views are generated by wrappers/plugins, not maintained as canonical mega-files.
5. OpenCode integration should be a bridge around `todowrite`, not a replacement for it.

## Non-goals

- Do not store secrets, full transcripts, or raw debug logs in task sidecars.
- Do not require every todo.txt client to understand agent-specific metadata.
- Do not mix permanent project rules with volatile task/session state.
